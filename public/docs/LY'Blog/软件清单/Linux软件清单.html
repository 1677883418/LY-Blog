    vuePress-theme-reco 肥羊    2021 - 2022

    Linux软件清单

    肥羊 7/20/2022 ArchLinux软件清单

    # 前言

    个人的一点经验,可以帮你选择合适的Linux发行版,但是也夹杂着一大堆废话，不想看的话可以直接跳过

    # Linux发行版

    我个人从高二开始使用Linux,我的使用历程大概分为桌面端和服务端两方面

    # 服务端

    # 起因

    • 我最早接触Linux,是我家里有个很拉垮的电脑，我想用那个电脑开MC服务器和朋友联机,为什么不用windows开服呢?因为配置真的很拉垮,配置如下:
      • AMD双核处理器速龙 X4 640(原装是X2 220,后来花了几十块钱换的)
      • ddr2 双内存条 4G*2(没错,是ddr2,开始2G,后来慢慢换到了8G)
      • 512MB的亮机卡(后来换成750ti还玩了狗2) 我用这个电脑度过了愉快的时光,一开始只能玩我的世界纯净服,后来可以加一些模组,再后来换了一台e3 1230 v2 这个时候我学会了Linux操作，frp内网穿透以及一些配置文件的编辑等技能,所以我一直认为,兴趣是最好的老师

    # 历程

    • 最开始的时候使用的发行版是Centos，一开始别人都在用，教程(鸟哥的Linux私房菜,强烈推荐！！！)也都是Centos，所以就跟着用
    • 后来了解的多了慢慢受不了Centos高安全性的代价，比如开web服务器需要关SELinux,很多软件包yum都没有，于是转到了ubuntu做服务端，后来慢慢改到用Debian，一直到现在还是用Debian

    目前Centos已经宣布转为滚动更新版本,以后的Centos也不再安全稳定(可能?),我个人推荐服务端使用Debian,低占用,软件包够用

    # 桌面端

    • 桌面端的开始当然是从'全球领先的用于个人电脑、平板及手机的操作系统'ubuntu开始的(ubuntu官网原话),中间下载错了，下到了某国产盗版Linux操作系统优麒麟(也可能是银河麒麟)，之前软件商店都没办法使用，刚刚去查了一下发现目前发展的很棒，UI深得我心.
    颜值才是第一生产力

    后来折腾折腾终于下到了Ubuntu,但是早期的ubuntu是使用unity桌面的，后来换到Gnome桌面,我个人觉得太素了不好看(不会美化)，于是寻寻觅觅换到了deepin，这也是我后来使用最久的Linux操作系统.放图

    DDE yyds

    # Deepin

    deepin我大概用了两年的时间，是我当初认为最好的操作系统不可否认deepin正处于早期，但是我认为他值得一试尤其是颜控,

    • 优点
      • 美观，易用，国内各种软件都可以开箱即用
      • 从windows无痛迁移，可以帮助适应linux系统
      • DDE配套很多原创软件,符合使用习惯
    • 缺点
      • 缺点也很明显，最主要的就是容易卡顿(截至2022-10-10,deepin目前主要版本为20.7，日常体验上没有bug,11年的联想老笔记本除了无线网卡无法驱动,没有任何问题)
      • DDE占用较高虽然颜值在线,但是相较于其他桌面环境,DDE的占用仍然较高,这也是造成卡顿的主要原因
      • 不方便个性化定制这也是我最终转战Arch的主要原因,Deepin以Ubuntu作为上游,大部分软件版本相对较老,内核、软件包等等,一方面代表了稳定,但是另一方面就代表了落后,很多新特性没办法体验
    Deepin的管理界面

    综合来说,deepin仍然是我认为最好的国产操作系统,最起码deepin真的有在做认真做操作系统,做了自己的桌面DDE,做了很多原生软件,深度相机,深度截图等等,包括Arch下使用的微信和钉钉也是从deepin移植的,

    # openSUSE

    deepin之后我用过openSUSE,德国人开发的系统,似乎和KDE是同一批开发人员.可以获得KDE原汁原味的体验

    • 优点:
      • 两种版本,长期支持版本与滚动更新版本,用户可以根据自己需要选择合适的版本安装
      • 因为是KDE原开发人员开发,理论上来说图形界面bug较少
    • 缺点:
      • 系统默认包管理器zypper非常劝退德国人做的系统比Centos还安全,加个镜像源超级麻烦

    # Arch

    后来开始用Arch,一直到现在都是,pacman+AUR基本可以满足所有的需求

    • 优点
      • pacman安装软件很舒服，AUR包永远不会让你失望,滚动系统永远都是最新的每天都有可能滚挂
      • 最好的ArchWiki,堪称百科全书
    • 缺点
      • 安装没有图形界面(但是能够很深刻的理解安装一个操作系统需要什么步骤)
      • 偶尔会出现各种各样的bug,有些很影响体验,需要重装(大部分情况是安装的软件太多冲突导致的)
    最近才重装，简单美化一下

    # 参考资料

    言归正传,Arch安装时没有图形界面以前我也没有记录的习惯，因此我每次重装主要参考以下网站:

    • ArchWiki:linux最全的官方wiki,大部分问题都可以在上面找到答案
      • ArchWiki (opens new window)
    • Arch 安装指南:一套很全的安装教程,大部分常用软件也可以在里面找到，推荐使用 似乎已经被墙了
      • Arch 安装指南 (opens new window)
    • Arch论坛:类似于贴吧？时常有大佬混迹其中,一般来说一个帖子对应一个问题,找不到的问题直接发报错截图询问即可。
      • Arch 中文论坛 (opens new window)

    # 常用软件

    下文中的paru请自行替换为自己的AUR助手,如yay等

    # markdown编辑

    • obsidian:用于形成个人知识图谱，写markdown文件只是最基础的功能
    sudo pacman -S obsidian
    1

    # 镜像源管理

    • Mirrorlist Manager:Arch镜像源管理工具,没有中文
    paru -S mirrorlist-manager
    1

    # Grub启动项编辑

    • grub-customizer:图形化编辑启动项,包括换主题,修改启动项等,有中文
    sudo pacman -S grub-customizer
    1

    # 控制中心

    • tuxedo-control-center:控制风扇转速，设置电源计划等等,适用于蓝天,清华同方等工模笔记本(其实是德国厂商给自己产品定制的控制中心)没有中文，不过影响不大
    paru -S tuxedo-control-center-bin tuxedo-keyboard
    1

    如果风扇转速显示灰色,大概率是缺少内核header,结合个人内核下载header包 linux内核:

    sudo pacman -S linux-headers
    1

    linux-zen:

    sudo pacman -S linux-zen-headers
    1

    # 音乐

    # yesplayMusic

    网易云三方客户端,颜值很高,使用web端API,

    • 优点: 可以在设置内换源达到解锁灰色歌曲的目的
    • 缺点: 使用Web端API时不时卡歌影响体验可能是我网络不佳

    # 网易云音乐

    sudo pacman -S netease-cloud-music
